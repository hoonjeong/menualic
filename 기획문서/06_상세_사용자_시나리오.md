# 메뉴얼 제작 서비스 - 상세 사용자 시나리오

## 시나리오 1: 신규 팀장이 팀을 만들고 첫 메뉴얼을 작성하는 경우

### 배경
김팀장은 새로 구성된 마케팅팀의 리더로, 팀원들과 업무 프로세스를 체계적으로 공유하고 싶어합니다.

### 목표
- 팀을 생성하고 팀원을 초대
- 첫 메뉴얼 "마케팅 업무 가이드" 작성
- 팀원들에게 공유

### 상세 단계

**1단계: 회원가입 및 팀 생성**
- 김팀장은 서비스 홈페이지 접속
- "시작하기" 버튼 클릭
- 이메일, 비밀번호, 이름 입력하여 회원가입
- 가입 완료 후 "팀 생성하기" 안내 페이지 표시
- 팀명 "마케팅팀" 입력
- 팀 설명 "2025년 신규 마케팅팀" 입력
- "팀 생성" 버튼 클릭
- **결과**: 소유자 권한으로 대시보드 진입

**2단계: 팀원 초대**
- 대시보드 좌측 사이드바에서 "팀 관리" 클릭
- "팀원 초대" 버튼 클릭
- 이메일 주소 3개 입력:
  - park@company.com (편집자)
  - lee@company.com (편집자)
  - choi@company.com (뷰어)
- 각 팀원의 권한 설정
- "초대 메일 발송" 버튼 클릭
- **결과**: 3명의 팀원에게 초대 메일 발송, 초대 대기 목록에 추가됨

**3단계: 첫 메뉴얼 작성 시작**
- 대시보드로 돌아가기
- 좌측 사이드바에서 "새 메뉴얼 추가" 버튼 클릭
- 메뉴얼 제목 "마케팅 업무 가이드" 입력
- 편집 화면으로 자동 전환
- **결과**: 비어있는 편집기 표시

**4단계: 계층 구조 만들기**
- 좌측 계층 트리에서 "대제목 추가" 클릭
- "1. 팀 소개" 입력
- 다시 "대제목 추가" 클릭하여 "2. 주요 업무" 입력
- "2. 주요 업무" 우클릭 → "하위 섹션 추가"
- "2-1. 콘텐츠 제작" 입력
- "2-1. 콘텐츠 제작" 우클릭 → "하위 섹션 추가"
- "2-1-1. 블로그 작성 가이드" 입력
- **결과**: 3단계 계층 구조 완성

**5단계: 콘텐츠 작성 (텍스트 + 이미지)**
- "1. 팀 소개" 섹션 클릭
- 중앙 편집 영역에서 "텍스트 블록 추가" 클릭
- "우리 팀은 브랜드 인지도 향상을 목표로 합니다" 입력
- 서식 툴바에서 "굵게" 적용
- "이미지 블록 추가" 클릭
- 팀 조직도 이미지 드래그앤드롭
- 이미지 크기 80%로 조정, 중앙 정렬
- 캡션 "2025년 마케팅팀 조직도" 입력
- **결과**: 텍스트와 이미지가 조합된 섹션 완성

**6단계: 블록 순서 변경**
- 이미지 블록 좌측 드래그 핸들 클릭
- 텍스트 블록 위로 드래그
- 파란색 드롭 라인 확인 후 드롭
- **결과**: 이미지가 텍스트보다 위로 이동

**7단계: 다른 섹션 작성**
- "2-1-1. 블로그 작성 가이드" 섹션 클릭
- 텍스트 블록 추가하여 작성 프로세스 입력
- 번호 목록 서식 적용
  1. 주제 선정
  2. 초안 작성
  3. 검토 요청
  4. 발행
- **결과**: 구조화된 가이드 완성

**8단계: 자동 저장 확인**
- 우측 하단에 "마지막 저장: 방금 전" 표시 확인
- 5초마다 자동 저장 아이콘 확인
- **결과**: 변경사항 자동 보존

**9단계: 팀원에게 공유**
- 우측 설정 패널에서 "공유 설정" 탭 클릭
- "팀 내부 공유" 선택
- 팀원 3명 모두 체크
- 박 대리, 이 대리: 편집자 권한
- 최 인턴: 뷰어 권한
- "공유" 버튼 클릭
- **결과**: 팀원 3명의 대시보드에 메뉴얼 표시, 각자에게 알림 발송

**10단계: 편집 종료**
- 상단 "나가기" 버튼 클릭
- 최종 저장 확인 모달 → "확인"
- 대시보드로 복귀
- **결과**: 메뉴얼이 좌측 사이드바 목록에 표시됨

### 최종 성과
- ✅ 팀 생성 완료
- ✅ 3명의 팀원 초대 완료
- ✅ 첫 메뉴얼 작성 및 공유 완료
- ✅ 팀원들이 각자 권한에 맞게 메뉴얼 접근 가능

## 시나리오 2: 팀원이 초대받아 메뉴얼을 편집하는 경우

### 배경
박 대리는 김팀장으로부터 초대 이메일을 받았고, 편집자 권한으로 메뉴얼에 내용을 추가하려고 합니다.

### 목표
- 초대 수락 및 팀 가입
- 기존 메뉴얼에 새로운 섹션 추가
- 이미지와 텍스트 조합하여 콘텐츠 작성

### 상세 단계

**1단계: 초대 수락**
- 박 대리는 이메일에서 "팀 초대 수락" 버튼 클릭
- 회원가입 페이지로 이동
- 이메일(자동 입력됨), 비밀번호, 이름 입력
- 회원가입 완료
- **결과**: 자동으로 "마케팅팀"에 편집자 권한으로 가입, 대시보드 진입

**2단계: 메뉴얼 확인**
- 좌측 사이드바에서 "마케팅 업무 가이드" 확인
- 메뉴얼 클릭하여 내용 읽기
- 김팀장이 작성한 내용 확인
- **결과**: 기존 메뉴얼 내용 숙지

**3단계: 편집 모드 진입**
- 우측 상단 "편집" 버튼 클릭
- 편집 화면으로 전환
- **결과**: 편집자 권한이므로 수정 가능

**4단계: 새 섹션 추가**
- 좌측 계층 트리에서 "2. 주요 업무" 우클릭
- "하위 섹션 추가" 선택
- "2-2. SNS 마케팅" 입력
- **결과**: 새로운 대제목 추가됨

**5단계: 콘텐츠 작성**
- "2-2. SNS 마케팅" 섹션 클릭
- 텍스트 블록 추가
- "SNS 플랫폼별 가이드" 제목 입력 (H2 서식)
- 또 다른 텍스트 블록 추가
- "인스타그램: 시각 중심 콘텐츠" 입력 (H3 서식)
- 세부 내용 작성
- **결과**: 구조화된 텍스트 콘텐츠 완성

**6단계: 이미지 추가**
- 이미지 블록 추가
- 인스타그램 성과 리포트 이미지 파일 선택
- 이미지 업로드 완료
- 크기 100%로 설정
- 캡션 "2024년 12월 인스타그램 성과" 입력
- **결과**: 이미지 블록 추가 완료

**7단계: 블록 순서 조정**
- 이미지 블록을 텍스트 블록 아래로 드래그
- 드롭 위치 확인 후 드롭
- **결과**: 텍스트 → 이미지 순서로 배치

**8단계: 자동 저장 및 버전 이력**
- 5초 후 자동 저장 알림 확인
- 우측 패널 "버전 이력" 탭 클릭
- 방금 전 저장된 버전 확인 (작성자: 박 대리)
- **결과**: 변경 사항 버전 이력에 기록됨

**9단계: 편집 완료**
- "나가기" 버튼 클릭
- 대시보드로 복귀
- 김팀장에게 "메뉴얼이 수정되었습니다" 알림 발송
- **결과**: 편집 내용 반영, 팀원들이 업데이트된 메뉴얼 확인 가능

### 최종 성과
- ✅ 팀 가입 완료
- ✅ 기존 메뉴얼에 새 섹션 추가
- ✅ 이미지와 텍스트 조합하여 전문적인 콘텐츠 작성
- ✅ 버전 이력 자동 기록

## 시나리오 3: 뷰어 권한 팀원이 메뉴얼을 보고 권한 요청하는 경우

### 배경
최 인턴은 뷰어 권한으로 팀에 가입했지만, 메뉴얼을 수정하고 싶어합니다.

### 목표
- 메뉴얼 읽기
- 편집 권한 부족 확인
- 팀장에게 권한 변경 요청

### 상세 단계

**1단계: 메뉴얼 열람**
- 최 인턴은 초대 링크로 회원가입 및 팀 가입 완료
- 대시보드에서 "마케팅 업무 가이드" 클릭
- 메뉴얼 내용 읽기
- **결과**: 읽기 전용 모드로 메뉴얼 확인

**2단계: 편집 시도**
- 메뉴얼 화면에서 "편집" 버튼 확인
- "편집" 버튼 비활성화 상태
- 호버 시 툴팁 "편집 권한이 없습니다" 표시
- **결과**: 편집 불가 확인

**3단계: 권한 변경 요청**
- 김팀장에게 메시지 또는 이메일 발송
- "편집 권한을 부여해주세요" 요청
- **결과**: 팀장의 권한 변경 대기

**4단계: 팀장의 권한 변경**
- 김팀장은 "팀 관리" 메뉴 접속
- 팀원 목록에서 "최 인턴" 선택
- 권한 드롭다운에서 "편집자"로 변경
- "변경" 버튼 클릭
- 최 인턴에게 "권한이 편집자로 변경되었습니다" 알림 발송
- **결과**: 최 인턴의 권한 업그레이드

**5단계: 편집 권한 확인**
- 최 인턴은 알림 확인
- 메뉴얼 다시 열람
- "편집" 버튼 활성화 확인
- 편집 모드 진입 가능
- **결과**: 이제 메뉴얼 수정 가능

### 최종 성과
- ✅ 뷰어 권한의 제약 이해
- ✅ 권한 변경 프로세스 경험
- ✅ 편집자로 업그레이드하여 협업 참여

## 시나리오 4: 외부 공유 링크로 비회원이 메뉴얼을 보는 경우

### 배경
김팀장은 외부 협력사와 메뉴얼을 공유하고 싶어합니다.

### 목표
- 외부 공유 링크 생성
- 비회원이 제목만 보도록 설정
- 회원가입 유도

### 상세 단계

**1단계: 외부 공유 설정**
- 김팀장은 "마케팅 업무 가이드" 편집 화면 진입
- 우측 설정 패널에서 "공유 설정" 탭 클릭
- "외부 공유" 선택
- 공유 범위 "제목만 공개" 선택
- "링크 생성" 버튼 클릭
- **결과**: 공유 링크 생성됨 (예: https://manual.service/share/abc123)

**2단계: 링크 복사 및 전달**
- "링크 복사" 버튼 클릭
- 이메일 또는 메신저로 협력사에 전달
- **결과**: 협력사가 링크 수신

**3단계: 비회원 접속**
- 협력사 담당자 (비회원)가 링크 클릭
- 메뉴얼 페이지 접속
- 좌측 사이드바에 계층 구조 제목 목록 표시
  - 1. 팀 소개
  - 2. 주요 업무
    - 2-1. 콘텐츠 제작
    - 2-2. SNS 마케팅
- 제목 클릭 시 내용은 흐리게 처리, "내용을 보려면 로그인하세요" 안내
- **결과**: 제목만 확인 가능, 회원가입 유도

**4단계: 회원가입 유도**
- 상단 배너 "전체 내용을 보려면 회원가입이 필요합니다" 표시
- "회원가입" 버튼 클릭
- 회원가입 페이지로 이동
- **결과**: 회원가입 후 전체 내용 접근 가능

### 최종 성과
- ✅ 외부 공유 링크 생성
- ✅ 비회원에게 제목 공개로 흥미 유발
- ✅ 회원가입 전환 유도

## 시나리오 5: 메뉴얼 수정 후 버전 되돌리기

### 배경
이 대리가 메뉴얼을 수정했는데, 실수로 중요한 내용을 삭제했습니다.

### 목표
- 이전 버전 확인
- 삭제되기 전 버전으로 복원

### 상세 단계

**1단계: 문제 발견**
- 이 대리는 "2-1-1. 블로그 작성 가이드" 섹션 수정 중
- 실수로 전체 섹션 삭제
- 자동 저장으로 삭제된 상태 반영
- **결과**: 중요한 가이드 내용 손실

**2단계: 버전 이력 확인**
- 우측 설정 패널에서 "버전 이력" 탭 클릭
- 버전 목록 확인
  - 5분 전: 이 대리 - "섹션 삭제"
  - 30분 전: 박 대리 - "이미지 추가"
  - 1시간 전: 김팀장 - "초기 작성"
- **결과**: 삭제 전 버전 확인 가능

**3단계: 버전 비교**
- "5분 전" 버전과 "30분 전" 버전 선택
- "비교" 버튼 클릭
- 삭제된 내용이 빨간색 취소선으로 표시
- **결과**: 삭제된 내용 정확히 파악

**4단계: 버전 되돌리기**
- "30분 전" 버전 선택
- "되돌리기" 버튼 클릭
- 확인 모달 표시
  - "이 버전으로 되돌리시겠습니까?"
  - "현재 내용은 새 버전으로 저장됩니다"
- "확인" 클릭
- **결과**: "30분 전" 상태로 복원

**5단계: 복원 확인**
- 편집 화면 새로고침
- "2-1-1. 블로그 작성 가이드" 섹션 복구 확인
- 삭제된 내용 모두 복원됨
- 버전 이력에 "복원됨" 표시 추가
- **결과**: 실수 복구 완료

### 최종 성과
- ✅ 버전 이력 활용하여 실수 복구
- ✅ 데이터 손실 방지
- ✅ 안전한 협업 환경 확보

---

## 시나리오 6: 팀 소유자가 팀원 관리하는 경우

### 배경
김팀장은 팀원 권한을 조정하고, 더 이상 팀에 속하지 않는 팀원을 내보내려고 합니다.

### 목표
- 팀원 권한 변경
- 팀원 내보내기
- 새로운 팀원 추가 초대

### 상세 단계

**1단계: 팀 관리 접속**
- 김팀장은 대시보드에서 "팀 관리" 클릭
- 팀원 목록 확인
  - 박 대리 (편집자)
  - 이 대리 (편집자)
  - 최 인턴 (편집자) ← 권한 업그레이드됨
- **결과**: 현재 팀원 상태 파악

**2단계: 팀원 내보내기**
- 이 대리가 퇴사하여 팀에서 제거 필요
- 이 대리 옆 "내보내기" 버튼 클릭
- 확인 모달 "정말 내보내시겠습니까?" 표시
- "확인" 클릭
- **결과**: 이 대리 팀 접근 권한 즉시 삭제, 작성한 메뉴얼은 팀에 유지

**3단계: 새 팀원 초대 (링크 방식)**
- "팀원 초대" 버튼 클릭
- 이번에는 "링크 초대" 선택
- 권한 "편집자" 선택
- 유효기간 "7일" 선택
- "링크 생성" 버튼 클릭
- 생성된 링크 복사
- **결과**: 초대 링크 생성, 새 팀원에게 전달 준비

**4단계: 링크 전달 및 가입 대기**
- 김팀장은 메신저로 신입 사원에게 링크 전달
- 신입 사원이 링크 클릭하여 회원가입
- 가입 즉시 팀에 편집자 권한으로 자동 추가
- 김팀장에게 "새 팀원이 가입했습니다" 알림
- **결과**: 신입 사원 팀 가입 완료

**5단계: 권한 재조정**
- 최 인턴이 업무에 익숙해짐
- 최 인턴 권한 유지 (편집자)
- 박 대리에게 추가 책임 부여 고려
- **결과**: 팀 구성 최적화

### 최종 성과
- ✅ 팀원 내보내기로 불필요한 접근 차단
- ✅ 링크 초대로 간편하게 신입 추가
- ✅ 권한 관리로 역할 명확화

## 시나리오별 핵심 인사이트

### 시나리오 1 (신규 팀장)
- **학습 포인트**: 팀 생성부터 메뉴얼 작성, 공유까지 전체 플로우
- **사용자 감정**: 처음엔 어색하지만, 드래그앤드롭 방식으로 빠르게 적응
- **개선 필요**: 첫 사용자를 위한 온보딩 가이드

### 시나리오 2 (편집자 팀원)
- **학습 포인트**: 초대 수락 및 협업 참여
- **사용자 감정**: 편집자 권한으로 적극적 참여 가능
- **개선 필요**: 편집 충돌 방지 메커니즘

### 시나리오 3 (뷰어 권한)
- **학습 포인트**: 권한 제약 및 변경 프로세스
- **사용자 감정**: 뷰어 권한 제약에 답답함, 권한 변경 후 만족
- **개선 필요**: 권한 요청 기능 (버튼으로 즉시 요청)

### 시나리오 4 (외부 공유)
- **학습 포인트**: 외부 공유 링크 생성 및 비회원 접근
- **사용자 감정**: 회원가입 없이도 일부 내용 확인 가능하여 편리
- **개선 필요**: 제목만 보기에서 더 많은 프리뷰 제공

### 시나리오 5 (버전 관리)
- **학습 포인트**: 버전 이력 활용한 실수 복구
- **사용자 감정**: 실수 복구로 안심, 버전 관리 신뢰
- **개선 필요**: 버전 비교 시각화 강화

### 시나리오 6 (팀 관리)
- **학습 포인트**: 팀원 추가/제거 및 권한 관리
- **사용자 감정**: 소유자로서 팀 통제 가능
- **개선 필요**: 팀원 활동 대시보드 (누가 얼마나 기여했는지)
